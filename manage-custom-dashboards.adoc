= Benutzerdefinierte Dashboards verwalten
:allow-uri-read: 
:icons: font
:imagesdir: ./media/


[role="lead"]
Verwalten Sie Ihre benutzerdefinierten Dashboards im Dashboard-Katalog. Dort lassen sich Details anzeigen, Daten aktualisieren oder Dashboards, die nicht mehr benötigt werden, löschen.

.Über diese Aufgabe
Alle Benutzer Ihres Workload Factory Kontos haben Zugriff auf alle benutzerdefinierten Dashboards. Jedes Dashboard zeigt grundlegende Details wie den Namen des Erstellers und das Erstellungsdatum.

Benutzerdefinierte Dashboards sind schreibgeschützt. Sie können weder Ihre Speicher- noch Ihre Workload-Einstellungen ändern.

.Bevor Sie beginnen
Sie müssen mindestens ein benutzerdefiniertes Dashboard in Ihrem Workload Factory Konto haben.



[[access-the-dashboard-catalog]]
== Auf den Dashboard Katalog zugreifen

.Schritte
. Melden Sie sich mit einem der link:console-experiences.html["Konsolenerfahrungen"^] an.
. Im Menü *Workloads Overview* und anschließend *Custom dashboards* auswählen.
+
Der Dashboard Katalog zeigt alle benutzerdefinierten Dashboards in Ihrem Workload Factory Konto an.

. Die Dashboard-Liste wird angezeigt.




[[refresh-dashboard-data]]
== Dashboard-Daten aktualisieren

Dashboard-Daten werden beim Öffnen eines Dashboards geladen. Eine Aktualisierung zeigt die neuesten Informationen an.

Das Dashboard lädt Daten aus Ihren Workloads neu. Die Zeitangabe „Letzte Aktualisierung“ ändert sich und zeigt an, wann die Daten aktualisiert wurden.



[[delete-a-dashboard]]
== Ein Dashboard löschen

Dashboards, die nicht mehr benötigt werden, können gelöscht werden. Dieser Vorgang ist endgültig und kann nicht rückgängig gemacht werden.

.Schritte
. Auf der Seite *Benutzerdefinierte Dashboards* das Dashboard suchen, das entfernt werden soll.
. Im Dropdown-Menü *Aktionen* für dieses Dashboard die Option *Löschen* auswählen.
. Bestätigen Sie die Löschung.


.Ergebnis
Das Dashboard wird dauerhaft aus Ihrem Workload Factory-Konto entfernt und alle Benutzer verlieren den Zugriff darauf.
